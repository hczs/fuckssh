# fuckssh 开发计划

## MVP 分析

### 核心用户流程

1. **主路径（场景 A）**：用户拿到新 VPS 的 IP/用户/密码 → 运行 `fuckssh add` → 选择密码连接 → 填写信息 → 工具生成密钥、部署公钥、写入 `ssh config` → 用户执行 `ssh <别名>` 免密登录。
2. **次路径（场景 B）**：远端已能用私钥登录，仅缺 config → `fuckssh add` 密钥模式 → 写入 `Host` / `IdentityFile` → `ssh <别名>` 可用。
3. **管理路径（场景 C）**：`fuckssh list` 浏览全部 Host → `fuckssh search <关键词>` 按别名/IP/hostname 模糊定位。

### MVP 必须功能

| 功能 | PRD 优先级 | 说明 |
|------|------------|------|
| 列出 VPS | P0 | 解析 `~/.ssh/config`，表格输出 |
| 模糊搜索 VPS | P0 | 匹配别名、HostName、IP |
| 新 VPS 向导（密码连接） | P0 | 生成密钥 + 部署公钥 + 写 config |
| 新 VPS 向导（密钥连接） | P0 | 仅写 config，引用已有私钥 |
| 修改 config 前备份 | 安全要求 | `config.fuckssh.bak.<timestamp>` |
| 系统 `ssh` 检测与安装指引 | 技术选型 | 尤其 Windows |

### 技术栈约束

- **Go 1.22+**、**Cobra**、**huh**（向导）
- 密钥：**Ed25519 默认**，`x/crypto/ssh` OpenSSH marshal，不依赖 `ssh-keygen`
- 密码部署：`golang.org/x/crypto/ssh` 客户端
- config：**自研受限解析器**（逐 `Host` 块；`HostName`/`User`/`Port`/`IdentityFile`）
- 无服务端、无数据库；仅标准 `~/.ssh` 文件

### 已有基础

脚手架已完成（见 [fuckssh-scaffold-plan.md](./fuckssh-scaffold-plan.md)）：目录结构、Cobra 子命令占位、`go test`/`make lint`/CI 可用。开发计划从 **业务实现** 开始，不再重复搭骨架。

### 排期原则（与技术选型 §3.4 / 架构 ADR-5 对齐）

**先只读、后写入**：`list`/`search` → `ssh` 检测 → `keys` → `add`（密钥模式 → 密码模式）→ 文档与发布打磨。

## 验收路线图

| 阶段完成后 | 如何验证 |
|------------|----------|
| Phase 1 | 在本机真实 `~/.ssh/config` 上执行 `list` 与 `search`，结果与手工对照一致 |
| Phase 2 | 临时改 PATH 或在没有 ssh 的环境看到分平台安装文案 |
| Phase 3 | 生成密钥后 `ssh-keygen -l -f` 可读；Unix 权限 0600 |
| Phase 4 | 密钥模式走完：config 有新 Host，`ssh <别名>` 能连（需远端已接受该钥） |
| Phase 5 | 密码模式对新 VPS：`ssh <别名>` 免密；失败时能看到备份路径 |
| Phase 6 | 公司 Win + 家里 Mac 各走一遍主路径；`go test`/`lint` 绿 |
