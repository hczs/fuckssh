1. 架构概述

架构风格：模块化单体 CLI（Modular Monolith CLI）— 单二进制、无服务端、仅操作标准 ~/.ssh 文件。

不采用微服务、数据库或自有配置格式，以保证与 Tabby、VS Code、系统 ssh 互通，并控制单人项目复杂度。

3. 数据流设计

密码连接一站式配置

sequenceDiagram
    actor U as 用户
    participant W as wizard
    participant K as keys
    participant C as config
    participant S as sshclient
    participant V as VPS
    U->>W: fuckssh add
    W->>S: 检测 ssh
    W->>C: 备份 config
    W->>K: 生成密钥
    W->>C: 写入 Host
    W->>S: 部署 authorized_keys
    S->>V: SSH 密码登录
    W-->>U: ssh 别名

状态：无持久运行时状态；密码仅驻内存；配置数据仅存于 ~/.ssh。

异步处理：不适用。

4. CLI 命令契约

无 HTTP API；对外契约为 Cobra 子命令。

命令说明
fuckssh add新 VPS 向导（密码/密钥模式）
fuckssh list列出 Host 条目
fuckssh search <q>模糊搜索 alias/hostname/IP

退出码

码含义
1用户输入无效
2config 解析失败
3文件 IO 失败
4远端部署失败

7. 安全设计

  • 密码不持久化、不进日志
  • 私钥 Unix 0600；Windows ACL 封装于 platform
  • 修改 config 前自动备份
  • 禁止 shell 拼接；exec.Command 传参列表
  • V2 备份：口令 + AEAD，文档提示同步盘风险

8. 可观测性

无应用监控栈。默认 stdout 结果、stderr 错误；可选 --verbose。质量靠单元测试与双机冒烟。

9. 异常与容错

  • 解析错误带行号与片段
  • 部署失败：提示备份路径，不自动删密钥
  • 无 ssh：可完成部署后提示安装再验收
  • 远端写 authorized_keys 可重试 1～2 次

10. 关键决策记录

ADR-1 模块化单体 CLI — 匹配 PRD，无服务端。
ADR-2 Go 生成密钥 — 须用 OpenSSH marshal API。
ADR-3 自研受限 config 解析 — MVP 控制范围。
ADR-4 实现顺序 list/search → 检测 → keys → add。
